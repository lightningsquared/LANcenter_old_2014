History of the project

In 2003, a project was created called Autonomous LAN Party. It was a solid attempt to bring a system like this to LAN parties everywhere. Unfortunately, it was abandoned around 2008, leaving organizers without a modern solution for managing LAN parties.

We've always found it odd that while LAN parties often have the latest in high-tech PC hardware, the management of certain parts of these events are still done with pen and paper. We want to change that.

Technical Overview

LANcenter is built using Ruby on Rails, one of the biggest MVC frameworks for building Web applications. The MVC architecture allows for code to be separated depending on its purpose.

On the front end, we are using the Bootstrap CSS & JavaScript framework for the user interface. This decision allows us to more quickly build out a user interface without spending time on designing individual user interface components.

This project also uses the Challonge! API to store and manage tournaments. LANcenter specifically makes use of the challonge-ruby-gem library to more easily interact with the API.
